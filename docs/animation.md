# 自動ガイドアニメーション仕様（メモ）

プレイヤーが介入しなくても、特定の出来事をきっかけに自動で再生されるチュートリアル演出の仕様。
序盤（主にステージ1）で操作を教え、以降は新しいギミックが出るたびに最小限のガイドを出す。

## ロボのキャラクター・話し方

吹き出しでしゃべる「ロボ」の性格・口調。セリフを考えるときの基準にする。

- **立ち位置**: プレイヤーの相棒。いっしょに冒険する、頼もしい仲間。
- **口癖**: 「グレート」（うまくいったとき・ほめるときに使う）。
- **口調**: 明るく前向きで、ぐいぐい引っ張る頼れる存在。プレイヤーを**名前で呼ぶ**（セリフ内では `{playerName}` と書き、表示時に実際の名前へ差し替える）。
- **言葉づかい**: 子ども向けに、やさしい言葉とひらがな中心。むずかしい専門用語は避け、必要なら言いかえる。
- **トーン**: 失敗しても責めずに励ます。前へ進む後押しをする。

## ふりがな・文章ルール

吹き出しのセリフはプレイヤーに見せる文章なので、CLAUDE.md のルールに従う。

- **漢字にはふりがな（ルビ）を振る。**
- **小学生で習わない漢字は使わず、ひらがなにする**（小学校で習う漢字はふりがな付きで使ってよい）。
- カード名・ボタン名など画面の UI ラベルは、画面表示に合わせた表記を使う（例: 「たたかう」「じっこう」「ぼうぎょ」）。
- ふりがな記法: `漢字《ふりがな》`。熟語は語のまとまりで `専用《せんよう》` のように書く。表示時にルビとして描画する。

## アニメーションの種類

演出は次の3種類。1つのトリガーで複数を組み合わせて再生することもある。

| 種類 | 内容 | 用途 |
|---|---|---|
| **ロボの吹き出し** | ロボのセリフ・説明文を吹き出しで表示する（短い文を1つずつ。クリック／タップで次の吹き出しへ進む） | 説明・セリフ全般 |
| **指差し誘導系** | 押してほしいUI要素（ボタン・アイコン等）を指し示す／ハイライトする | 操作を促す |
| **再生アニメーション** | 画面上で何かが起こる演出（ステージ開放など） | 状態変化の表現 |

## 実装フェーズ

演出は段階的に作る。まずは一番わかりやすい吹き出しから。

- **フェーズ1（最初に作る）**: **ロボの吹き出しのみ**。`bubble` を持つ step だけを実装・表示する。
- **フェーズ2以降（あとで）**: 指差し誘導系（`point`）。指差しアニメーションは分かりづらいため後回し。
- **フェーズ2以降（あとで）**: 再生アニメーション（`playAnimation`、ステージ開放演出など）。

フェーズ1での扱い:

- `point` を持つガイド（巻物・たたかうボタン・マップなどを指す）は、指差しを出さず**吹き出しのテキストだけで誘導する**。
- 吹き出しを持たないガイド（#12 ステージ3開放 / #15 ステージ4開放）は、フェーズ1では何も表示しない（再生アニメはフェーズ2以降）。
- 各ガイドの詳細にある「指す対象」「再生アニメ」は、フェーズ2以降の参照用に残しておく。

## 吹き出しのレイアウト（表示位置）

吹き出しの出る場所は画面ごとに異なる。

### マップ画面
- **吹き出し**: 画面の**下側**に表示する。
- **ロボのアイコン**: **左上**に「フローチャートロボ」の画像をアイコンとして置く。

### 戦闘画面
- **吹き出し**: `enemyArea`（敵の表示エリア）の**左側**に置く。

## データ管理

演出データは `frontend/src/data/cutscenes.json` で一元管理する。

- **ロボの吹き出し文言・指差し誘導・再生アニメーション** は、チュートリアルかどうかに関係なく共通の演出部品として扱う。
- データは「**トリガー → 演出の並び（steps）**」という1つの形に統一する。`tutorial` / `talk` の区別は `category` タグで持つ（ファイルは分けない）。
- 各 step は次の3種類のキーを持てる。同じ step に併記すれば同時再生、step を分ければ順番に再生。
- **吹き出し（`bubble`）を持つ step は1つずつ表示し、プレイヤーのクリック／タップで次の step へ進む。** 短い文を複数の step に分けることで、テンポよく送れる。
- `point` を併記した step は、その吹き出しを表示している間だけ対象を指し示す。`playAnimation` の step はアニメ再生後に次へ進む。
- `bubble` 内の `{playerName}` は、表示時にプレイヤー名へ差し替える（ロボはプレイヤーを名前で呼ぶ）。
- `bubble` の漢字は `漢字《ふりがな》` 記法でルビを持たせる（→「ふりがな・文章ルール」参照）。

| キー | 対応する演出 | 内容 |
|---|---|---|
| `bubble` | ロボの吹き出し | 表示するセリフ・説明文（短い1文。クリックで次の step へ） |
| `point` | 指差し誘導系 | 指し示すUI要素のID（任意） |
| `playAnimation` | 再生アニメーション | 再生する演出名（例: `unlockStage`）＋ `params`（任意） |

```jsonc
// cutscenes.json（構造イメージ）
{
  "1-1-enter": {
    "category": "tutorial",
    "once": true,
    "trigger": { "type": "enterBattle", "stageId": "1-1" },
    "steps": [
      { "bubble": "まずは 敵《てき》の HP を 見《み》てみよう！", "point": "enemyHpBar" },
      { "bubble": "カードを スロットに 入《い》れてね",   "point": "cardSlot" }
    ]
  },
  "1-4-exit": {
    "category": "tutorial",
    "trigger": { "type": "exitStage", "stageId": "1-4" },
    "steps": [
      { "playAnimation": "unlockStage", "params": { "world": "2" } },
      { "bubble": "ステージ2が 開《ひら》いたよ！次《つぎ》の マップへ 行《い》こう！", "point": "mapButton" }
    ]
  },
  "title-greeting": {
    "category": "talk",
    "once": false,
    "trigger": { "type": "openScreen", "screen": "title" },
    "steps": [
      { "bubble": "やあ！ぼうけんを 始《はじ》めよう！" }
    ]
  }
}
```

## ガイドを出す/出さない判定

ガイドの表示は次の流れで判定する。

1. ゲーム内でイベントが起きる（ステージに入る、敵を倒す、ステージから出る、など）。
2. そのイベントに `trigger` が一致する cutscene を `cutscenes.json` から探す。
3. 見つかったら、表示済みかどうかを判定する。
   - `once: true`（デフォルト。チュートリアル向け）… すでに見たことがあれば**出さない**。まだなら出す。
   - `once: false`（あいさつなど繰り返すもの）… 毎回出す。
4. 出したら、その cutscene の `id` を「見たことがある」記録に追加する。

### 表示済みの記録（再表示しない仕組み）

- 新しいストア（例: `cutsceneStore`）で、見たガイドの `id` を `seenIds` として保持する。
- リロードしても再表示しないよう **localStorage に永続化**する。
  - 注: 既存の進行状態 `progressStore` はまだ永続化していないため、この仕組みはここで新しく入れる必要がある。

### トリガーの種類（例）

| type | 発火タイミング | 主なパラメータ |
|---|---|---|
| `enterBattle` | バトル画面に入る | `stageId` |
| `defeatEnemy` | 敵を倒す（勝利） | `stageId` |
| `battleLost` | バトルに負ける（#6 の失敗誘導用） | `stageId` |
| `exitStage` | クリアしてマップへ戻る | `stageId` |
| `arriveLandmark` | マップでランドマークに到着 | `stageId` |
| `openScreen` | 画面を開く | `screen` |

### 補足

- 「ガイドなしのステージ」は、一致する `trigger` の cutscene を用意しなければ何も出ない（出さないための特別な処理は不要）。
- 各ガイドの「表示条件」欄は、基本は `trigger` と `once` で表現する。さらに細かい条件が要る場合だけ追記する。

## 補足・未確定事項

- ガイドなしのステージ（2-1, 2-2, 3-2, 3-3, 4-1, 4-2 など）の扱い: → 当面はガイドを用意しない（cutscene を作らなければ何も出ない）。必要になったら後から追加する。
- 一度見たガイドの再表示有無: → `once` フラグ＋`seenIds`（localStorage 永続化）で管理。デフォルト `once: true`（一度見たら再表示しない）。
- 吹き出し文言のふりがな方式: → `漢字《ふりがな》` 記法でルビを振る。小学生で習わない漢字はひらがなにする（解決済み）。
