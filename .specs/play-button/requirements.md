# 要件定義: 実行ボタン（ビジュアル進行のみ）

## はじめに

フローチャートに配置したカードを **「実行する」**動作の起点となる UI とアニメーション機能。スタートマーカーからゴールマーカーまで、エッジを辿ってハイライトが順に進行する視覚的な演出を実装し、配置されたカード（スロット）を点滅させて「処理されている」実感を与える。本スペックでは **ビジュアル進行のみ**を対象とし、ダメージ計算・HP 増減・勝敗判定・敵の反撃などの戦闘ロジックは含まない（将来別スペックで段階的に追加する想定）。対象利用者はステージをプレイするプレイヤー本人。

## 要件

### 要件1: 実行ボタンの表示

**ユーザーストーリー：** プレイヤーとして、配置したフローチャートを実行する起点となる UI が欲しい。なぜならカードを配置するだけでは「動く感」が無く、押して結果を見る楽しみが必要だから。

#### 受け入れ基準

1. WHEN 戦闘画面でフローチャートが描画される THEN the system SHALL `ZoomButton` と `ResetButton` の **直下** にコンパクトな細長い実行ボタンを配置する
2. WHEN 実行ボタンが描画される THEN the system SHALL ボタン内にビデオ再生ボタン風の緑色アイコン（右向き三角形の左にすき間を空けて縦の直線を 1 本配置し、その結果として「左辺に二重線が走る」ように見せる構成）を SVG で表示する
3. WHEN 実行ボタンが描画される THEN the system SHALL ボタンの横幅を `ZoomButton` ＋ `ResetButton` の合計幅程度に揃え、上下のパディングは控えめにして「細長い」見た目にする

### 要件2: 全スロット埋まり時のみ有効化

**ユーザーストーリー：** プレイヤーとして、配置が未完成の状態では実行できないようにしてほしい。なぜなら部分配置で実行できると、空きスロットの扱いがあいまいになり、「ちゃんと組んでから実行」というゲームルールが弱まるから。

#### 受け入れ基準

1. IF フローチャート上に 1 つでもカード未配置のスロットがある場合 WHEN 実行ボタンが描画される THEN the system SHALL ボタンを無効状態（クリック不可）にする
2. IF 全てのスロットがカードで埋まっている場合 WHEN 実行ボタンが描画される THEN the system SHALL ボタンを有効状態（クリック可能）にする
3. WHILE 実行ボタンが無効状態 the system SHALL ボタン外観を「押せない」と分かる見た目（透明度を下げる、ホバー反応を止める、カーソルを `not-allowed` に）にする
4. IF 実行ボタンが無効状態 WHEN ユーザーが押そうとする THEN the system SHALL クリックを無視する（実行を開始しない）

### 要件3: 実行中の操作ロック

**ユーザーストーリー：** プレイヤーとして、実行中は誤操作で進行が壊れないようにしてほしい。なぜなら実行は一連のアニメーション処理であり、途中での状態変更は予期せぬ挙動の温床になるから。

#### 受け入れ基準

1. WHILE 実行中 the system SHALL カードのドラッグ＆ドロップ（手札→スロット、スロット→スロット、スロット→外への撤回 を含む全て）を受け付けない
2. WHILE 実行中 the system SHALL リセットボタン・ズームボタン・実行ボタン本体（連打防止）の押下を受け付けない
3. WHEN 実行が完了する THEN the system SHALL 上記のロックを全て解除し、通常の操作可能状態に戻す

### 要件4: 拡大状態からの自動縮小

**ユーザーストーリー：** プレイヤーとして、実行中は敵モンスターも視野に入る状態で進行を眺めたい。なぜなら実行は最終的に敵にダメージを与える流れにつながるので、敵の様子も視覚に入っているほうが演出として伝わるから。

#### 受け入れ基準

1. IF フローチャートが拡大状態である場合 WHEN ユーザーが実行ボタンを押す THEN the system SHALL 実行開始前にフローチャートを縮小状態に自動で戻す
2. WHEN 実行ボタン押下による自動縮小が走る THEN the system SHALL 既存の拡大／縮小アニメーション（0.25 秒のトランジション）と同じ動きを再利用する
3. WHEN 自動縮小アニメーションが完了する THEN the system SHALL 続けて実行アニメーションを開始する
4. IF フローチャートが既に縮小状態である場合 WHEN ユーザーが実行ボタンを押す THEN the system SHALL 縮小アニメーションをスキップして即座に実行を開始する

### 要件5: ハイライト進行のシーケンス

**ユーザーストーリー：** プレイヤーとして、フローチャートが実際にどう進むかを視覚的に追えるようにしたい。なぜなら配置の意味（スタートからゴールへの一本道）が動きで強化されると、ゲームのフィードバックが直感的になるから。

#### 受け入れ基準

1. WHEN 実行が開始される THEN the system SHALL スタートマーカーをハイライトすることから始める
2. WHEN スタートマーカーのハイライトが終わる THEN the system SHALL スタートからの最初のエッジを伝うように、光の点が線に沿って次のスロットまで進むアニメーションを再生する
3. WHEN 光の点が次のスロットに到達する THEN the system SHALL そのスロット上のカードを点滅させる（複数回明滅して「処理されている」感を出す）
4. WHEN スロット上での点滅が終わる THEN the system SHALL 次のエッジに沿って光の点を進める。これをゴールに到達するまで繰り返す
5. WHEN 光の点がゴールマーカーに到達する THEN the system SHALL ゴールマーカーをハイライトする
6. WHEN ゴールマーカーのハイライトが終わる THEN the system SHALL 全てのハイライトを解除し、実行を完了する

### 要件6: タイミング配分

**ユーザーストーリー：** プレイヤーとして、配置したカード数に応じて実行のテンポが変わってほしい。なぜならカードが多ければ「処理してる感」を時間で感じたいし、少なければサクッと終わってほしいから。

#### 受け入れ基準

1. WHEN 実行が開始される THEN the system SHALL 実行全体の所要時間 T を「配置済みカード数 N × 2 秒」として算出する（要件 2 により実行時は常に N = 全スロット数）
2. WHEN 実行が進行する THEN the system SHALL 算出した T を、`スタートのハイライト` ＋ `各エッジ通過` ＋ `各カード点滅` ＋ `ゴールのハイライト` の全フェーズに等分配する
3. WHEN 各フェーズの時間が決まる THEN the system SHALL ノード（マーカー・スロット）でのハイライト／点滅は静的に光るのではなく、フェーズ時間内で複数回明滅するアニメーションにする

### 要件7: 実行完了後の状態

**ユーザーストーリー：** プレイヤーとして、実行が終わった後はもう一度自由に配置をいじれる状態に戻ってほしい。なぜなら実行は「ビジュアル演出のみ」であり、配置や手札がそれによって変わるべきではない（ダメージ計算等は将来追加）から。

#### 受け入れ基準

1. WHEN 実行が完了する THEN the system SHALL 全てのハイライト・点滅・光の点アニメーションを停止し、見た目を実行前の状態に戻す
2. WHEN 実行が完了する THEN the system SHALL カードの配置（`slotAssignments`）と手札（`handCards`）を実行前の状態と同一に保つ（実行による変更なし）
3. WHEN 実行が完了する THEN the system SHALL リセットボタン・ズームボタン・実行ボタンを再度有効化する

### 要件8: 既存機能との整合性

**ユーザーストーリー：** プレイヤーとして、実行機能の追加で既存のリセット・配置 UI・拡大／縮小・スタート／ゴールマーカーが壊れないでほしい。

#### 受け入れ基準

1. WHEN フローチャート全体（スロット・マーカー・エッジ）が描画される THEN the system SHALL 既存の `card-placement` / `flowchart-zoom` / `flowchart-start-goal` スペックの仕様をすべて維持する
2. WHEN 実行が完了して通常状態に戻る THEN the system SHALL リセットボタン押下による配置初期化・拡大／縮小切替・カードのドラッグ&ドロップを既存仕様どおり動作させる
3. WHILE 実行中ではない（通常状態） the system SHALL 既存の挙動（手札からスロットへの配置、スロット間の入れ替え、リセット、拡大／縮小切替）を変更しない
