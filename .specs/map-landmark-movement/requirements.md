# 要件定義: マップ・ランドマーク移動

## はじめに

フィールドマップ上にプレイヤーキャラクターを表示し、マウスクリックでランドマーク間を移動できるようにする機能。主人公は初期状態でマップ西端の出発点ランドマーク（村の門）に立ち、プレイヤーが他のランドマーク（井戸・倒木・砦・ストーンサークルなど）をクリックすると、ランドマーク同士を結ぶ **道のネットワーク（分岐を含むグラフ）** 上の経路に沿って、キャラクターがアニメーションで移動する。ランドマークの座標と道のつながり（エッジ）はデータとして持ち、道は SVG + viewBox で描画する。

本スペックのスコープは「マップ画面の表示」「道のネットワーク上のランドマーク間移動」「移動アニメーション」までとし、**戦闘画面（BattleScreen）への遷移は含めない**。到着後のふるまいは後続スペックで扱う。

## 要件

### 要件1: マップ画面の表示

**ユーザーストーリー：** プレイヤーとして、ゲーム起動時にフィールドマップを見たい。なぜなら、どのランドマークに向かうか選びたいから。

#### 受け入れ基準

1. WHEN アプリが起動する THEN the system SHALL `App.jsx` が `BattleScreen` の代わりにマップ画面（MapScreen）を表示する
2. WHEN マップ画面が表示される THEN the system SHALL `public/maps/map_1.png` を背景として 16:9 のアスペクト比で描画する
3. WHEN マップ画面が表示される THEN the system SHALL 背景画像の上に道（SVG path）・ランドマーク・プレイヤーキャラクターを重ねて描画する

### 要件2: ランドマークと道のネットワーク

**ユーザーストーリー：** プレイヤーとして、マップ上のどこがランドマークで、どのランドマーク同士が道でつながっているのかを一目で把握したい。なぜなら、クリック可能な地点と移動経路を間違えずに選びたいから。

#### 受け入れ基準

1. WHEN マップ画面が表示される THEN the system SHALL データで定義された 5 つのランドマーク（村の門・井戸・倒木・砦・ストーンサークル）を、それぞれのアイコン座標（`position`）に描画する
2. WHEN マップ画面が表示される THEN the system SHALL データで定義されたランドマーク間のエッジ（道）を、両端ランドマークの停止点座標（`stopPoint`）を結ぶ SVG path として 1 本ずつ描画する
3. 各ランドマークは「アイコン・クリック判定の中心」となる `position` と、「道に接する停止点・パス端点」となる `stopPoint` の 2 つの座標を持つ。アイコンの見た目位置と道のラインを別軸で調整できるようにするため両者を分離する
4. WHEN マップ画面が表示される THEN the system SHALL ランドマーク同士の接続関係（どのランドマークとどのランドマークが道でつながっているか）を、データで定義された通りに反映する（特定のランドマークが分岐点になり得る）
5. IF ランドマークの座標・ID・エッジの接続関係を変更する必要が出た WHEN コードを編集する THEN the system SHALL JSON データファイルの編集のみで反映される構造になっている

### 要件3: 主人公の初期位置

**ユーザーストーリー：** プレイヤーとして、ゲームを始めた瞬間に自分のキャラクターがマップの出発点（村の門）にいることを確認したい。なぜなら、冒険の出発点がどこかが明確になるから。

#### 受け入れ基準

1. WHEN マップ画面が初回表示される THEN the system SHALL プレイヤーキャラクターを出発点ランドマーク（村の門）の座標に描画する
2. WHILE まだどのランドマークもクリックされていない the system SHALL プレイヤーキャラクターを出発点に静止させる
3. WHEN マップ定義で出発点 ID（`startId` 等）が指定されている THEN the system SHALL そのランドマークを初期位置として採用する（コード変更なしで初期位置を切り替えられる）

### 要件4: クリックによるランドマークへの移動

**ユーザーストーリー：** プレイヤーとして、行きたいランドマークをクリックして、自分のキャラクターをそこへ向かわせたい。なぜなら、マウスだけで直感的にマップを探索したいから。

#### 受け入れ基準

1. WHEN プレイヤーがランドマークをクリックする THEN the system SHALL そのランドマークを目的地として採用し、移動を開始する
2. IF クリックされたランドマークと現在地が道で直接つながっている（隣接している） WHEN クリックされる THEN the system SHALL その 1 本のエッジに沿ってキャラクターを移動させる
3. IF クリックされたランドマークと現在地が直接つながっていない WHEN クリックされる THEN the system SHALL 道のネットワーク上の経路（エッジの列）を辿って目的地まで移動させる。経由するランドマーク数が最小になる経路を選ぶ
4. IF プレイヤーが現在いるランドマークと同じランドマークをクリックする THEN the system SHALL 何も起こさない
5. IF キャラクターが移動中である WHEN プレイヤーがランドマークをクリックする THEN the system SHALL そのクリックを無視する（移動完了までクリック入力を受け付けない）

### 要件5: 移動アニメーション

**ユーザーストーリー：** プレイヤーとして、ランドマーク間を移動する様子をアニメーションで見たい。なぜなら、瞬間的にテレポートするより移動している実感が欲しいから。

#### 受け入れ基準

1. WHEN 移動が開始される THEN the system SHALL キャラクタースプライトがパス上を連続的に（瞬間移動せず）滑らかに動く
2. WHEN 移動が進行中 the system SHALL キャラクターの座標を SVG path に沿って補間する（パス長に対する進行割合 0→1 で位置を決める）
3. WHEN 移動が完了する THEN the system SHALL キャラクターを目的地ランドマークの停止点座標（`stopPoint`）に正確に着地させ、静止状態に戻す
4. WHILE 移動中 the system SHALL 移動方向の進捗が目視で分かるよう一定速度または滑らかなイージングで再生する（瞬間的にジャンプしない）
5. WHEN 経由ランドマークを含む移動が行われる THEN the system SHALL 経路上の各エッジを順に連続再生し、経由ランドマークで不自然に長時間停止しない

### 要件6: SVG + viewBox による道の生成

**ユーザーストーリー：** 開発者として、ランドマーク座標と接続関係から道を SVG で生成したい。なぜなら、画像をトレースせずデータ駆動で道を引けるほうが、配置変更やマップ追加に強いから。

#### 受け入れ基準

1. WHEN マップ画面が描画される THEN the system SHALL 背景画像と同じ座標系で SVG 要素を配置し、viewBox を使って論理座標（例：0–1920 × 0–1080）で道・ランドマーク・キャラクターを指定できるようにする
2. WHEN 道を描画する THEN the system SHALL ランドマーク同士を結ぶエッジごとに SVG `<path>` を 1 本ずつ生成する（エッジ数 = `<path>` 要素数）
3. IF ウィンドウサイズが変わった WHEN マップ画面が表示されている THEN the system SHALL SVG viewBox の仕組みにより背景画像・道・キャラクターの相対位置が崩れない
4. WHEN 移動アニメーションが走る THEN the system SHALL SVG path の `getPointAtLength()` などのパス追従 API を用いて、各エッジの形状に忠実にキャラクターを動かす（直線補間ではなく道のカーブに沿う）
