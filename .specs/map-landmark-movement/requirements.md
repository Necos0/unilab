# 要件定義: マップ・ランドマーク移動

## はじめに

フィールドマップ上にプレイヤーキャラクターを表示し、マウスクリックでランドマーク間を移動できるようにする機能。主人公は初期状態で中央ハブに立ち、プレイヤーがランドマーク（井戸・倒木・砦・ストーンサークルなど）をクリックすると、あらかじめ定義されたパスに沿ってキャラクターがアニメーションで移動する。ランドマークの座標はデータとして持ち、道は SVG + viewBox で描画する。

本スペックのスコープは「マップ画面の表示」「ハブを経由するランドマーク間移動」「移動アニメーション」までとし、**戦闘画面（BattleScreen）への遷移は含めない**。到着後のふるまいは後続スペックで扱う。

## 要件

### 要件1: マップ画面の表示

**ユーザーストーリー：** プレイヤーとして、ゲーム起動時にフィールドマップを見たい。なぜなら、どのランドマークに向かうか選びたいから。

#### 受け入れ基準

1. WHEN アプリが起動する THEN the system SHALL `App.jsx` が `BattleScreen` の代わりにマップ画面（MapScreen）を表示する
2. WHEN マップ画面が表示される THEN the system SHALL `public/maps/map_1.png` を背景として 16:9 のアスペクト比で描画する
3. WHEN マップ画面が表示される THEN the system SHALL 背景画像の上に道（SVG path）・ランドマーク・プレイヤーキャラクターを重ねて描画する

### 要件2: ランドマークとハブの配置

**ユーザーストーリー：** プレイヤーとして、マップ上のどこがハブで、どこがランドマークなのかを一目で把握したい。なぜなら、クリック可能な地点を間違えずに選びたいから。

#### 受け入れ基準

1. WHEN マップ画面が表示される THEN the system SHALL 中央のハブ地点と 4 つのランドマーク（北西：井戸、北東：倒木、南西：砦、南東：ストーンサークル）を、データで定義された座標に描画する
2. WHEN マップ画面が表示される THEN the system SHALL 各ランドマークとハブを結ぶ道（SVG path）を描画する
3. IF ランドマークの座標や ID を変更する必要が出た WHEN コードを編集する THEN the system SHALL JSON データファイルの編集のみで座標・ID が反映される構造になっている

### 要件3: 主人公の初期位置

**ユーザーストーリー：** プレイヤーとして、ゲームを始めた瞬間に自分のキャラクターがマップの中央ハブにいることを確認したい。なぜなら、冒険の出発点がどこかが明確になるから。

#### 受け入れ基準

1. WHEN マップ画面が初回表示される THEN the system SHALL プレイヤーキャラクターをハブの座標に描画する
2. WHILE まだどのランドマークもクリックされていない the system SHALL プレイヤーキャラクターをハブ位置に静止させる

### 要件4: クリックによるランドマークへの移動

**ユーザーストーリー：** プレイヤーとして、行きたいランドマークをクリックして、自分のキャラクターをそこへ向かわせたい。なぜなら、マウスだけで直感的にマップを探索したいから。

#### 受け入れ基準

1. WHEN プレイヤーがランドマークをクリックする THEN the system SHALL そのランドマークを目的地として採用し、移動を開始する
2. IF プレイヤーが現在ハブにいる WHEN 任意のランドマークをクリックする THEN the system SHALL ハブ → 対象ランドマークのパスに沿ってキャラクターを移動させる
3. IF プレイヤーが既に別のランドマークにいる WHEN 異なるランドマークをクリックする THEN the system SHALL 現ランドマーク → ハブ → 対象ランドマーク の順でパスを辿らせる（すべての移動はハブを経由する）
4. IF プレイヤーが現在いるランドマークと同じランドマークをクリックする THEN the system SHALL 何も起こさない
5. IF キャラクターが移動中である WHEN プレイヤーがランドマーク・ハブをクリックする THEN the system SHALL そのクリックを無視する（移動完了までクリック入力を受け付けない）
6. WHEN プレイヤーがハブをクリックする THEN the system SHALL プレイヤーが既にハブにいれば何もせず、別ランドマークにいれば現ランドマーク → ハブ のパスで移動する

### 要件5: 移動アニメーション

**ユーザーストーリー：** プレイヤーとして、ランドマーク間を移動する様子をアニメーションで見たい。なぜなら、瞬間的にテレポートするより移動している実感が欲しいから。

#### 受け入れ基準

1. WHEN 移動が開始される THEN the system SHALL キャラクタースプライトがパス上を連続的に（瞬間移動せず）滑らかに動く
2. WHEN 移動が進行中 the system SHALL キャラクターの座標を SVG path に沿って補間する（パス長に対する進行割合 0→1 で位置を決める）
3. WHEN 移動が完了する THEN the system SHALL キャラクターを目的地ランドマーク（またはハブ）の座標に正確に着地させ、静止状態に戻す
4. WHILE 移動中 the system SHALL 移動方向の進捗が目視で分かるよう一定速度または滑らかなイージングで再生する（瞬間的にジャンプしない）
5. WHEN ハブ経由のランドマーク間移動が行われる THEN the system SHALL 現ランドマーク→ハブ、ハブ→目的地 の 2 区間を連続再生し、ハブで不自然に長時間停止しない

### 要件6: SVG + viewBox による道の生成

**ユーザーストーリー：** 開発者として、ランドマーク座標から道を SVG で生成したい。なぜなら、画像をトレースせずデータ駆動で道を引けるほうが、配置変更やマップ追加に強いから。

#### 受け入れ基準

1. WHEN マップ画面が描画される THEN the system SHALL 背景画像と同じ座標系で SVG 要素を配置し、viewBox を使って論理座標（例：0–1920 × 0–1080）で道・ランドマーク・キャラクターを指定できるようにする
2. WHEN 道を描画する THEN the system SHALL ランドマーク座標とハブ座標を結ぶ SVG `<path>` を、各ランドマークごとに 1 本ずつ生成する
3. IF ウィンドウサイズが変わった WHEN マップ画面が表示されている THEN the system SHALL SVG viewBox の仕組みにより背景画像・道・キャラクターの相対位置が崩れない
4. WHEN 移動アニメーションが走る THEN the system SHALL SVG path の `getPointAtLength()` などのパス追従 API を用いて、道の形状に忠実にキャラクターを動かす（直線補間ではなく道のカーブに沿う）
