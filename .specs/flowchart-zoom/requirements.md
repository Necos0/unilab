# 要件定義: フローチャート拡大／縮小トグル

## はじめに

戦闘画面中段のフローチャート領域を、敵エリアを覆い隠すところまで上方向に広げられる「拡大／縮小トグル」機能。現在フローチャートは 1 行だが、将来 2〜3 行に拡張する予定であり、スロット数が増えると縮小状態では 1 スロットあたりの表示サイズが小さくなる。拡大状態では個別スロットを原寸（80×120）で確認・操作でき、縮小状態ではフローチャート全体を俯瞰しながら手札・敵との同時表示が保てる。対象利用者はステージをプレイするプレイヤー本人で、本スペックでは切替 UI・自動スケーリング・アニメーションまでを扱い、複数行フローチャートの配置定義（`stages.json` 側のデータ構造拡張）は含まない。

## 要件

### 要件1: 拡大／縮小トグルボタン

**ユーザーストーリー：** プレイヤーとして、フローチャートエリアの表示サイズを 1 クリックで切り替えたい。なぜなら「全体を俯瞰して置きたい」場面と「個別スロットを大きく見ながら置きたい」場面の両方があるから。

#### 受け入れ基準

1. WHEN 戦闘画面が表示される THEN the system SHALL フローチャート領域の右上（既存リセットボタンの左隣）に拡大／縮小トグルボタンを配置する
2. WHILE フローチャートが縮小状態 the system SHALL トグルボタンに上向き矢印「↑」を表示する
3. WHILE フローチャートが拡大状態 the system SHALL トグルボタンに下向き矢印「↓」を表示する
4. WHEN ユーザーがトグルボタンを押す THEN the system SHALL 縮小状態 ⇄ 拡大状態を切り替える
5. WHEN トグルボタンがリセットボタンと並んで描画される THEN the system SHALL 両ボタンの見た目（色・サイズ）を揃える

### 要件2: 縮小状態（デフォルト）

**ユーザーストーリー：** プレイヤーとして、戦闘画面を開いた直後は敵・フローチャート・手札をひと目で確認したい。なぜなら戦況を把握しながら戦略を組み立てる必要があるから。

#### 受け入れ基準

1. WHEN 戦闘画面が初めてマウントされる THEN the system SHALL フローチャートを縮小状態で表示する（初期値）
2. WHILE フローチャートが縮小状態 the system SHALL 敵エリア・フローチャートエリア・プレイヤーエリアの 3 段レイアウト（現在の 40 : 40 : 20 相当）を維持する
3. WHILE フローチャートが縮小状態 the system SHALL React Flow の自動フィットで全スロット＋エッジが縮小エリアに収まるよう表示する（余裕がある場合は React Flow 既定の上限 2.0 まで自動拡大、収まらない場合は 1.0 未満に自動縮小）

### 要件3: 拡大状態

**ユーザーストーリー：** プレイヤーとして、フローチャートを拡大して個別スロットを大きく見たい。なぜならスロット数が増えてくると縮小状態では 1 スロットが小さすぎて内容が見づらくなるから。

#### 受け入れ基準

1. WHEN フローチャートが拡大状態 THEN the system SHALL フローチャートエリアを上方向に広げ、敵スプライト・敵 HP バーを画面外（もしくは覆い隠された状態）にする
2. WHILE フローチャートが拡大状態 the system SHALL プレイヤーエリア（HP バー・手札）のサイズと位置を縮小状態と同一に保つ
3. WHILE フローチャートが拡大状態 the system SHALL スロットのスケール係数 1.0（原寸 80×120）を下限として、拡大エリアに収まる範囲で React Flow の自動フィットで拡大表示する（スケールは React Flow 既定の上限まで許容）
4. IF フローチャート全体がスケール 1.0 で拡大エリアに収まらない場合 WHEN 拡大状態で表示する THEN the system SHALL スケール 1.0 を維持し（1.0 未満への縮小は行わず）、あふれた部分はエリア内のスクロールでアクセスさせる

### 要件4: 切替アニメーション

**ユーザーストーリー：** プレイヤーとして、切替時にレイアウトがパッと跳ねるより滑らかに遷移してほしい。なぜなら視覚的に状態変化を追えるほうが操作への不安が少ないから。

#### 受け入れ基準

1. WHEN ユーザーがトグルボタンを押す THEN the system SHALL 0.2 〜 0.3 秒のスムーズなトランジションでレイアウトを変化させる
2. WHILE 切替中 the system SHALL フローチャートエリアの高さとスロットのスケール係数を同時に補間する
3. WHILE 切替中 the system SHALL リセットボタン・トグルボタンの位置（フローチャート領域右上）を視覚的に追従させる

### 要件5: 両状態でのドラッグ＆ドロップ維持

**ユーザーストーリー：** プレイヤーとして、縮小・拡大のどちらの状態でもカードをスロットにドラッグ＆ドロップで配置したい。なぜなら縮小状態でも配置・入替を軽くやりたいし、拡大状態では大きい UI で丁寧に操作したいから。

#### 受け入れ基準

1. WHILE フローチャートが縮小状態 the system SHALL 手札からスロット、スロットから別スロット、スロットから外（手札へ戻す）の全ドラッグ&ドロップを既存仕様どおり動作させる
2. WHILE フローチャートが拡大状態 the system SHALL 上記の全ドラッグ&ドロップを既存仕様どおり動作させる
3. WHILE 切替アニメーション中 the system SHALL ドラッグ&ドロップおよびリセット／トグルボタン押下など画面上の全ユーザー操作を受け付けない（バグ防止のためブロックする）
4. IF ユーザーが切替アニメーション中にポインタを押下／移動した場合 WHEN アニメーションが進行している THEN the system SHALL 当該操作を無視し、状態遷移を起こさない
5. IF フローチャートが縮小状態でスロットが自動縮小されている場合 WHEN ユーザーがスロット上でドロップ判定する THEN the system SHALL 縮小後のスロット描画範囲をヒットテスト領域として扱う

### 要件6: 既存機能との整合性

**ユーザーストーリー：** プレイヤーとして、拡大／縮小機能を追加してもリセット・配置・入れ替え・手札への撤回などの既存挙動が変わらないでほしい。なぜなら既に慣れた操作感を保ったまま新機能を使いたいから。

#### 受け入れ基準

1. WHEN ユーザーがリセットボタンを押す THEN the system SHALL 拡大／縮小状態を維持したままフローチャート上のカード配置のみを初期化する
2. WHEN カードが配置・入れ替え・撤回される THEN the system SHALL 拡大／縮小状態に関わらず既存の `battleStore` ロジックを変更なく実行する
3. WHILE フローチャートが拡大状態 the system SHALL 敵データ（`enemyId`・`maxHp` 等）をストアや親コンポーネントから消さず保持する（拡大解除時に即座に再描画できるように）
