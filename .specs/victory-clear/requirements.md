# 要件定義: 勝利時 CLEAR! 演出

## はじめに

フローチャート実行終了時点で敵 HP が 0 になっていたとき、戦闘画面上で **「敵を倒した → 勝利を確定する → マップへ戻る」** までの一連の流れを視覚的に演出する機能。敵スプライトの dead アニメーション（用意されている場合）→ スプライトのフェードアウト → 「CLEAR!」テキスト表示 → 「マップへ戻る」ボタンの誘導、という時系列で進行し、CLEAR! 表示中はマップへ戻る以外の操作をすべて無効化する。

これにより、プレイヤーは「フローチャートが期待どおり動いて敵を倒せた」という結果を達成感をもって受け取り、次に取るべきアクション（マップへ戻る）に迷わず誘導される。対象利用者はステージをプレイするプレイヤー本人。本スペックは攻撃処理（`attack-processing`）の延長に位置するが、敗北時の演出・複数ステージ間の進行管理・敵の反撃などは別スペックで扱う。

## 要件

### 要件1: 勝利判定のトリガー

**ユーザーストーリー：** プレイヤーとして、フローチャートを実行し終えたときに敵 HP が 0 になっていれば、勝利演出が自動的に始まってほしい。なぜなら「実行が終わった」「敵を倒せた」というゲーム上のマイルストーンを、プレイヤーが手動で確認しなくても明示的にフィードバックしてほしいから。

#### 受け入れ基準

1. WHEN フローチャートの実行シーケンスが完了する AND 敵 HP が 0 である THEN the system SHALL CLEAR! 演出シーケンスを開始する
2. WHEN フローチャートの実行シーケンスが完了する AND 敵 HP が 1 以上である THEN the system SHALL CLEAR! 演出を起動せず、これまでどおり通常状態へ戻す
3. IF 敵 HP が実行途中で 0 になっても WHEN 実行シーケンスが完了するまで THEN the system SHALL CLEAR! 演出を開始しない（実行終了の合図と同時にトリガーする）

### 要件2: dead アニメーションの再生（実装済み敵のみ）

**ユーザーストーリー：** プレイヤーとして、敵を倒した瞬間に「やられた」モーションが見たい。なぜなら HP バーが 0 になっただけでは討伐の達成感が薄く、敵スプライト自体に視覚的な変化が欲しいから。

#### 受け入れ基準

1. WHEN CLEAR! 演出シーケンスが開始される AND 倒した敵に dead 状態のスプライト定義が存在する THEN the system SHALL idle アニメーションから dead アニメーションへ切り替え、最終フレームまで 1 回だけ再生する（ループしない）
2. WHEN dead アニメーションが最終フレームに到達する THEN the system SHALL 最終フレームを停止表示したままフェードアウトフェーズへ進む
3. IF 倒した敵に dead 状態のスプライト定義が存在しない場合 WHEN CLEAR! 演出シーケンスが開始される THEN the system SHALL dead アニメーションをスキップし、idle 表示のままフェードアウトフェーズへ進む

> **備考**: 本要件 3 は将来の暫定措置。全敵に dead アニメーションを実装した時点で、スキップ分岐は撤去される可能性がある。

### 要件3: 敵スプライトのフェードアウト

**ユーザーストーリー：** プレイヤーとして、倒した敵が静かに消えていく演出を見たい。なぜなら一気に消えるのではなく徐々にフェードすることで、戦闘の終わりが穏やかに伝わり、CLEAR! 表示への視覚的な橋渡しになるから。

#### 受け入れ基準

1. WHEN dead アニメーション再生（または dead 未実装時はそのスキップ）が完了する THEN the system SHALL 敵スプライトを 0.5 秒程度かけて不透明度 1 → 0 へ徐々に変化させる
2. WHEN フェードアウトが完了する THEN the system SHALL 敵スプライトを完全に透明（または非表示）の状態で固定する
3. WHILE フェードアウトが進行中 the system SHALL スプライトの位置・サイズを変更しない（透過のみで動かない）

### 要件4: 「CLEAR!」テキストの表示

**ユーザーストーリー：** プレイヤーとして、勝利したことが画面で大きく告げられる達成感を得たい。なぜなら HP バーや敵スプライトの状態だけでなく、明示的な「CLEAR!」表記がゲーム的な区切りになるから。

#### 受け入れ基準

1. WHEN 敵スプライトのフェードアウトが完了する THEN the system SHALL 「CLEAR!」というテキストを敵エリア中央の **上半分** に表示する
2. WHEN CLEAR! テキストが描画される THEN the system SHALL HP バーの数値ラベル（プレイヤー／敵共通の `.hpText` 相当）と統一感のあるドット絵風ピクセルフォントを用い、HP バーの数値より十分に大きいサイズで強調表示する
3. WHILE CLEAR! テキストが表示されている the system SHALL 戦闘画面のレイアウト（敵エリア／フローチャートエリア／プレイヤーエリアの 3 分割）を維持し、敵エリア外には突き出さない

### 要件5: 「マップへ戻る」ボタンの再配置

**ユーザーストーリー：** プレイヤーとして、勝利後はマップへ戻ることが次の自然な行動になるので、その動線が画面中央に明示されてほしい。なぜなら勝利直後にどこを押せばいいか迷わず、「マップへ戻る」が唯一進められるアクションだと一目で分かるようにしたいから。

#### 受け入れ基準

1. WHILE CLEAR! 演出が表示されていない通常状態 the system SHALL 「マップへ戻る」ボタンを戦闘画面の右上（既存位置）に表示する
2. WHEN CLEAR! テキストが表示される THEN the system SHALL 「マップへ戻る」ボタンを敵エリア中央の **下半分**（CLEAR! テキストの直下）へ移動して表示する
3. WHEN 「マップへ戻る」ボタンが押される THEN the system SHALL 既存どおりマップ画面への遷移ハンドラ（`onExitToMap`）を呼び出す（CLEAR! 中も通常時も挙動は同じ）

### 要件6: CLEAR! 表示中の操作無効化

**ユーザーストーリー：** プレイヤーとして、勝利後は誤って実行ボタンやリセットボタンを押してしまわないようにしたい。なぜなら CLEAR! 中はマップへ戻ることだけが意味のある操作であり、誤操作で演出が中断したり HP がリセットされたりすると達成感が損なわれるから。

#### 受け入れ基準

1. WHILE CLEAR! 演出シーケンスが進行中（dead アニメ・フェードアウト・CLEAR! 表示）または CLEAR! テキストが表示されている the system SHALL カードのドラッグ＆ドロップ操作を一切受け付けない
2. WHILE CLEAR! 演出中 the system SHALL 実行ボタン（PlayButton）・リセットボタン（ResetButton）・拡大トグルボタン（ZoomButton）を無効化する
3. WHILE CLEAR! 演出中 the system SHALL 「マップへ戻る」ボタンのみクリック可能にする
4. IF プレイヤーが CLEAR! 中に画面のどこかをクリック・ドラッグしようとする WHEN ボタン以外の領域 THEN the system SHALL いかなる状態変更も発生させない

### 要件7: 既存機能との整合性

**ユーザーストーリー：** プレイヤーとして、CLEAR! 演出の追加によって既存の戦闘進行・HP バー・攻撃演出・拡大／縮小切替の挙動が損なわれないでほしい。

#### 受け入れ基準

1. WHEN ステージ初期化（`initializeBattle`）が走る THEN the system SHALL CLEAR! 状態をクリアし、敵スプライトを通常の不透明度 1・idle 状態に戻す
2. WHEN 攻撃処理（`attack-processing`）の演出（フラッシュ・ダメージ数字フロート・HP バー減少）が走る THEN the system SHALL CLEAR! 演出と干渉せず、これまでと同じタイミング・見た目で動作する
3. WHEN CLEAR! 演出が起動していない通常状態 the system SHALL 戦闘画面の見た目・操作・遷移を従来仕様（`attack-processing` / `play-button` / `flowchart-zoom` 等）と完全に一致させる
4. WHEN 「マップへ戻る」ボタンを押してマップへ戻った後、再び同じステージに入り直す THEN the system SHALL 敵スプライト・CLEAR! 状態・HP を初期化し、CLEAR! の名残が残らない
