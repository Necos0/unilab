# 要件定義: カード配置（ドラッグ＆ドロップ）

## はじめに

戦闘画面の右下に描画されている手札カードを、中段のフローチャート上の空きスロットへドラッグ＆ドロップで配置できるようにする機能。現状は手札とスロットが描画されるだけで両者が結びついておらず、プレイヤーが戦略を組み立てる余地がない。本機能によりカードの配置 UI が成立し、以降の「実行」や戦闘処理の土台となる。対象利用者はステージをプレイするプレイヤー本人。

本スペックはカード配置の UI 操作までを対象とし、実行ボタン・ダメージ計算・勝敗判定などの戦闘処理は別スペックで扱う。

## 要件

### 要件1: カードのドラッグ開始

**ユーザーストーリー：** プレイヤーとして、手札またはスロット上のカードを直感的につかんで動かしたい。なぜなら自分の意思でフローチャートの好きな位置にカードを配置・再配置したいから。

#### 受け入れ基準

1. WHEN ユーザーが手札のカードをポインタでプレスしてそのまま動かし始める THEN the system SHALL そのカードをドラッグ中状態にする
2. WHEN ユーザーがスロット上のカードをポインタでプレスしてそのまま動かし始める THEN the system SHALL そのカードをドラッグ中状態にする
3. WHILE カードがドラッグ中 the system SHALL ドラッグ中のカードをポインタ位置に追従させて描画する
4. WHILE カードがドラッグ中 the system SHALL ドロップ可能なスロット（空きスロットおよび他のカードが置かれたスロット）を視覚的にハイライトする
5. WHILE 手札のカードがドラッグ中 the system SHALL 手札上のその位置からカードを一時的に取り除いて（または半透明にして）「つかんでいる」ことを示す
6. WHILE スロット上のカードがドラッグ中 the system SHALL 元のスロットを空きスロット表示に切り替えて「つかんでいる」ことを示す

### 要件2: 空きスロットへの配置

**ユーザーストーリー：** プレイヤーとして、手札やスロットからつかんだカードを空いているスロットに置きたい。なぜならフローチャートを組み立てて戦略を決めたいから。

#### 受け入れ基準

1. WHEN ユーザーがドラッグ中のカードを空きスロット上でドロップする THEN the system SHALL そのスロットに当該カードを配置する
2. WHEN 手札からドラッグしたカードが空きスロットに配置される THEN the system SHALL そのカードを手札から取り除く（以降、手札には表示されない）
3. IF ドラッグ元が別のスロットである場合 WHEN ユーザーがドラッグ中のカードを空きスロットにドロップする THEN the system SHALL 元のスロットを空きスロット表示に戻す
4. WHEN カードがスロットに配置される THEN the system SHALL スロット上のカードを手札と同じ見た目（カード画像と `power` 数値）で描画する

### 要件3: 配置済みスロットへのドロップ（差し替え）

**ユーザーストーリー：** プレイヤーとして、既にカードが置かれているスロットに別のカードを置きたい。なぜなら戦略を変えたいときに少ない操作で差し替えたいから。

#### 受け入れ基準

1. WHEN ユーザーがドラッグ中のカードを既にカードが置かれているスロット上でドロップする THEN the system SHALL 元々そのスロットにあったカードを手札に戻す
2. WHEN ユーザーがドラッグ中のカードを既にカードが置かれているスロット上でドロップする THEN the system SHALL ドロップされたカードを当該スロットに配置する
3. IF ドラッグ元が別のスロットである場合 WHEN ユーザーがドラッグ中のカードを配置済みスロットにドロップする THEN the system SHALL ドラッグ元のスロットを空きスロット表示に戻す
4. IF ドラッグ元と同一のスロットに戻すようにドロップした場合 WHEN ユーザーがドロップを確定する THEN the system SHALL カードを元のスロットに据え置き、手札の状態も変更しない

### 要件4: スロット外へのドロップ（手札に戻す／キャンセル）

**ユーザーストーリー：** プレイヤーとして、ドラッグしたカードをスロット以外の場所で離したら手札に戻ってほしい。なぜならキャンセル操作と「スロットから手札への撤回」操作が同じ直感で扱えると操作負担が減るから。

#### 受け入れ基準

1. IF ドラッグ元が手札である場合 WHEN ユーザーがドラッグ中のカードをスロット外にドロップする THEN the system SHALL 当該カードを手札に残したままにする（状態変化なし）
2. IF ドラッグ元がスロットである場合 WHEN ユーザーがドラッグ中のカードをスロット外にドロップする THEN the system SHALL 当該カードを手札に戻す
3. IF ドラッグ元がスロットである場合 WHEN ユーザーがドラッグ中のカードをスロット外にドロップする THEN the system SHALL ドラッグ元のスロットを空きスロット表示に戻す

### 要件5: 同一カードの複数所持への対応

**ユーザーストーリー：** プレイヤーとして、手札に同じ種類のカード（例：`attack` ×2）があっても、それぞれを独立して 1 枚ずつ配置できるようにしたい。なぜなら将来ステージによっては同種カードが複数枚配られる設計になるから。

#### 受け入れ基準

1. IF 手札に同一 `id` のカードが複数枚含まれる場合 WHEN ユーザーがそのうち 1 枚をドラッグしてスロットに配置する THEN the system SHALL 配置した 1 枚のみを手札から取り除き、残りのカードは手札に残す
2. IF 手札に同一 `id` のカードが複数枚含まれる場合 WHEN 各カードをドラッグ対象として識別する THEN the system SHALL それぞれを別の個体として区別する（同じ `id` でも異なる個体として扱う）

### 要件6: 既存レイアウトとの両立

**ユーザーストーリー：** プレイヤーとして、ドラッグ＆ドロップが加わっても戦闘画面全体のレイアウトが崩れないでほしい。なぜなら敵や HP バーなどの既存 UI も同時に使うから。

#### 受け入れ基準

1. WHEN ユーザーがカードをドラッグする THEN the system SHALL フローチャート領域のパン・ズームなど React Flow 側の既存インタラクションを誘発しない
2. WHEN 画面サイズが既存の戦闘画面で想定された範囲内で変化する THEN the system SHALL 敵エリア・フローチャート領域・プレイヤー HP・手札の 3 段レイアウトを維持する
3. IF ステージ定義の `cards` または `slots` が空の場合 WHEN 戦闘画面を表示する THEN the system SHALL 従来通りエラーなく描画する（ドラッグ機能の有無で既存レンダリングが破綻しない）
