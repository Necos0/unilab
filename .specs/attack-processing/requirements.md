# 要件定義: 攻撃カード処理（ダメージ計算と演出）

## はじめに

フローチャート実行中に attack カードが配置されたスロットを通った際、その `card.power` 値を敵 HP から減算する戦闘ロジックを実装する。あわせて「攻撃が当たった」ことを伝える演出（敵スプライトのフラッシュ・ダメージ数字のフロート表示・HP バーの滑らかな減少）と、敵 HP バーへの数値併記（プレイヤー HP バーと同等の `current/max` 表示）を加える。

このゲームは **「プログラミングを書く → 実行 → 結果を観察 → デバッグして修正 → 再実行」** という思考の反復を学習する目的を持つ。そのため敵 HP は **実行のたびにリセット** され、各実行が独立した「コードのテスト実行」として成立する設計とする。本スペックは攻撃処理に関する戦闘ロジックと演出までを対象とし、勝敗判定・勝利／敗北演出・敵の反撃・guard / heal カードの効果は別スペックで扱う。対象利用者はステージをプレイするプレイヤー本人。

## 要件

### 要件1: 敵 HP の状態管理

**ユーザーストーリー：** プレイヤーとして、敵に与えたダメージが累積した結果として現在の敵 HP が把握できるようにしたい。なぜなら自分のフローチャート（コード）が期待通りの結果（敵 HP の到達値）を出しているかを確認することが、プログラミング学習サイクルの根幹だから。

#### 受け入れ基準

1. WHEN 戦闘画面がマウントされる THEN the system SHALL 敵 HP を `enemies.json` に定義された `maxHp` 値で初期化する
2. WHILE 戦闘画面が表示されている the system SHALL 現在の敵 HP を `battleStore` でグローバルに保持し、HP バー描画と数値表示で同じ値を参照する
3. IF ダメージ計算結果として敵 HP が 0 を下回る WHEN ダメージが適用される THEN the system SHALL 敵 HP を 0 にクランプし、マイナス値にしない

### 要件2: 攻撃カードによるダメージ適用

**ユーザーストーリー：** プレイヤーとして、フローチャート実行中に attack カードを配置したスロットを通ったとき、そのカードの `power` 値が敵に与えるダメージとして反映されてほしい。なぜなら配置したカードの値が結果に直結することで、「どのカードを置けばいくつのダメージになるか」というプログラミング的因果関係を学べるから。

#### 受け入れ基準

1. WHEN 実行シーケンス進行中にスロットノードがハイライトフェーズに入る AND そのスロットに `id === 'attack'` のカードが配置されている THEN the system SHALL そのカードの `power` 値を敵 HP から減算する
2. IF 配置されているカードが `attack` 以外（`guard` / `heal` 等）の場合 WHEN スロットノードがハイライトフェーズに入る THEN the system SHALL the system SHALL ダメージを適用しない（HP は変化しない）
3. WHEN ダメージが適用される THEN the system SHALL ハイライトフェーズの開始タイミングに合わせて減算を実行し、フラッシュ演出・ダメージ数字フロート・HP バー減少の各演出を時系列で同期させる

### 要件3: 実行ごとの HP リセット

**ユーザーストーリー：** プレイヤーとして、実行ボタンを押すたびに敵 HP が初期値（`maxHp`）からスタートしてほしい。なぜならこのゲームは「コードを書く→実行→結果を見る→修正→再実行」のループを通じて目的の動作を作り上げる **デバッグ駆動の学習体験** であり、各実行はフレッシュな状態から走らないと「フローチャートを修正したらどう結果が変わるか」を比較できないから。

#### 受け入れ基準

1. WHEN ユーザーが実行ボタンを押し、実行シーケンスが開始される THEN the system SHALL 敵 HP を `maxHp` に戻してからシーケンスを開始する
2. WHEN 実行シーケンスが完了する THEN the system SHALL 敵 HP をシーケンス終了時点の値（0〜maxHp）で保持する（次の実行が始まるまで動かさない）
3. WHILE 実行終了後〜次の実行開始までの通常状態 the system SHALL 敵 HP の現在値を可視化したまま維持し、ユーザーが「今回の実行でいくつまで削れたか」を確認できる
4. IF ユーザーがリセットボタンを押す（カード配置の初期化）WHEN 通常状態 THEN the system SHALL 敵 HP には触れない（HP のリセットは次の実行開始時に自動で起きるため、リセットボタンの責務は配置のみ）

### 要件4: 敵 HP バーの数値表示

**ユーザーストーリー：** プレイヤーとして、敵 HP の現在値と最大値を数値で正確に把握したい。なぜなら「あと何ダメージで倒せるか」をプログラミング的に逆算する必要があり、視覚的なバーの長さだけでは正確な計算がしづらいから。

#### 受け入れ基準

1. WHEN 敵 HP バーが描画される THEN the system SHALL `currentHp/maxHp` 形式の数値ラベルをバーに併記する（プレイヤー HP バーと同じレイアウト）
2. WHEN 敵 HP が変化する THEN the system SHALL バーと数値ラベルが共に同期して更新される
3. WHEN 数値ラベルが描画される THEN the system SHALL プレイヤー HP バーで使われている既存の `.hpText` 相当のスタイル（フォント・色・サイズ）を流用し、見た目の一貫性を保つ

### 要件5: 敵スプライトのフラッシュ演出

**ユーザーストーリー：** プレイヤーとして、攻撃が敵に当たった瞬間に「ヒットした」感を視覚的に感じたい。なぜなら HP バーの減少だけでは「いつどのカードが効いたか」を時系列で追いづらいから。

#### 受け入れ基準

1. WHEN attack カードのスロットがハイライトフェーズに入りダメージが適用される THEN the system SHALL 敵スプライトを白っぽくフラッシュさせる演出を起動する
2. WHEN フラッシュ演出が走る THEN the system SHALL 0.2 〜 0.3 秒の短時間で 1 回または 2 回明滅して自然に終わる
3. WHILE フラッシュ中 the system SHALL 敵スプライトの位置・サイズ・既存 idle アニメーションを変更しない（演出は重ね描き）

### 要件6: ダメージ数字のフロート表示

**ユーザーストーリー：** プレイヤーとして、ダメージ量を数値ですぐに確認したい。なぜなら「12 与えた」「3 与えた」とその場で見えると、置いたカードの `power` と結果の対応が直感的にわかり、デバッグ思考に寄与するから。

#### 受け入れ基準

1. WHEN attack カードのスロットがハイライトフェーズに入りダメージが適用される THEN the system SHALL 敵スプライトの近くに「-<damage>」形式（例：「-12」）の数字を出現させる
2. WHEN ダメージ数字が表示される THEN the system SHALL 上方向にゆっくり浮き上がりながら徐々に透明になり、0.6 〜 1.0 秒程度で完全に消える
3. IF 同じ実行シーケンス内で複数の attack カードが連続する場合 WHEN 各ダメージが順次適用される THEN the system SHALL 各々のダメージ数字を独立した要素として描画し、重なって見えてもアニメーションが互いに干渉しない

### 要件7: 敵 HP バーのアニメーション

**ユーザーストーリー：** プレイヤーとして、HP バーの減り方が一瞬でカクッと変わるより、少しでも滑らかに変化してほしい。なぜなら短時間でも「減った感」が視覚で伝わると、ヒットの手応えが増してフィードバックがリッチになるから。

#### 受け入れ基準

1. WHEN 敵 HP が減少する THEN the system SHALL HP バーの幅変化を 0.2 〜 0.3 秒のトランジションで滑らかに変化させる
2. WHEN HP の数値ラベルも変化する THEN the system SHALL バーの幅変化と同時に数値が更新される（バーが減るのと同時に数字も減る）

### 要件8: 既存機能との整合性

**ユーザーストーリー：** プレイヤーとして、攻撃処理が加わっても既存のカード配置・実行・リセット・拡大／縮小の挙動が変わらないでほしい。

#### 受け入れ基準

1. WHEN ユーザーがリセットボタンを押す THEN the system SHALL 既存仕様どおり手札とスロット割当を初期化する（敵 HP には触れない、要件 3-4）
2. WHEN ユーザーが拡大トグルを押す、カードをドラッグする等の通常操作を行う THEN the system SHALL 敵 HP を変更しない
3. WHEN 実行が完了して通常状態に戻る THEN the system SHALL カードの配置（`slotAssignments`）と手札（`handCards`）を実行前の状態と同一に保つ（既存の play-button 仕様 7-2 を維持）
4. WHEN フローチャートの拡大／縮小切替が走る THEN the system SHALL 敵 HP バーの数値表示・フラッシュ演出・ダメージ数字も既存のレイアウトを崩さず動作する
