# 要件定義: フローチャートの描画

## はじめに

戦闘画面中段のフローチャート領域に、ステージごとに定義された空きスロットと
それらを接続するエッジ（線）を描画する機能。この時点ではスロットは空き枠として
表示されるのみで、カードの配置やドラッグ&ドロップは含まない。後続のスペックで
カード配置ロジック・手札 UI・バリデーション等を追加する土台として実装する。

## スコープ

### 含まれるもの
- フローチャートエリアへのスロット（空き枠）の描画
- スロット間を繋ぐエッジ（線）の描画
- スロット位置・エッジ・ステージ情報の静的データ化

### 含まれないもの
- カード・手札・ドラッグ&ドロップ（別スペック）
- ステータスエリアと手札エリアの分割（別スペック）
- スロットへのインタラクション（クリック・ホバー等）
- エッジの自動レイアウト計算（位置は事前定義）
- ステージ切り替え UI（1 ステージ固定で表示）

## 要件

### 要件1: 空きスロットの描画
**ユーザーストーリー：** プレイヤーとして、戦闘画面中段に配置される空きスロットを
視覚的に認識したい。なぜなら、ここにカードを入れるのだと直感的に理解できる
必要があるから。

#### 受け入れ基準
1. WHEN 戦闘画面が表示される THEN the system SHALL フローチャートエリア内に定義された全スロットを空き枠として描画する
2. WHEN スロットが描画される THEN the system SHALL 各スロットが「カードを置く場所」であることが視覚的に分かる外観（点線枠や空のカード型など）を持つ
3. WHEN スロット定義に位置情報が与えられる THEN the system SHALL フローチャートエリア内の指定位置にスロットを配置する

### 要件2: スロット間の接続線（エッジ）の描画
**ユーザーストーリー：** プレイヤーとして、スロット同士がどう繋がっているかを
線で見たい。なぜなら、処理の流れる順序を把握するにはフローが可視化されている
必要があるから。

#### 受け入れ基準
1. WHEN 戦闘画面が表示される THEN the system SHALL 定義された全エッジをスロット間に線として描画する
2. WHEN エッジが描画される THEN the system SHALL 接続の向き（始点から終点）が分かる表現（矢印など）を含む
3. IF エッジの始点または終点のスロットが存在しない WHEN 描画が要求される THEN the system SHALL そのエッジを描画せず、他の描画には影響を与えない

### 要件3: ステージデータの事前定義
**ユーザーストーリー：** 開発者として、スロットとエッジの配置をデータファイルで
定義したい。なぜなら、ステージを増やすときにコード変更を最小化したいから。

#### 受け入れ基準
1. WHEN 本スペックの実装時点 THEN the system SHALL スロット・エッジの定義を JSON ファイルとして `frontend/src/data/` 配下に保持する
2. WHEN ステージデータが読み込まれる THEN the system SHALL スロットの ID・位置と、エッジの始点 ID・終点 ID を取り出せる
3. WHEN 本スペックの実装時点 THEN the system SHALL 動作確認用に 1 ステージ（スロット 3〜4 個、エッジでシンプルに接続）のみを定義する
